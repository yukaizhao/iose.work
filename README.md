# 面向意图软件工程

《面向意图软件工程》是一本探讨 AI 时代软件工程范式转变的系统指南与开源电子书。本书系统探讨从传统软件工程向「面向意图软件工程」（IOSE）过渡的完整路径，为技术团队提供切实可行的转型方案。无论你是技术管理者、架构师、开发者，还是创业者，都能从中获得关于未来软件工程的深刻洞见和实操指南。

## 阅读建议

1. **渐进式阅读**：本书按「理解→过渡→实践→展望」的递进逻辑编排，建议按顺序阅读。
2. **结合实践**：每章配有实操模板和检查清单，建议边读边在团队中尝试。
3. **持续迭代**：面向意图软件工程仍在快速发展，保持开放心态，持续调整认知。

## 核心概念速览

| 概念 | 定义 |
|------|------|
| **意图（Intent）** | 对「要什么」和「为什么」的清晰表达，而非「怎么做」 |
| **业务意图** | 顶层业务目标，描述价值诉求和商业逻辑 |
| **约束网络** | 从被动文档到可执行代码的五维质量空间（架构、复杂度、安全、性能、可维护性），作为人机协同的「质量契约」与 Agent 的「实时教科书」 |
| **Agent 工程师** | 从执行层跃迁到调度层，作为「首席架构师」指挥 Agent 集群交付复杂系统的技术角色 |

## 核心观点

- **意图分层**：业务意图定义「要什么」，技术意图定义「边界在哪里」。
- **范式转变**：从「写代码」到「定义意图」的根本转变。
- **人机协作**：人类负责意图，AI 负责实现。
